# CognID-GenRec 目前项目进展

> 最近更新：2026-07-11  
> 当前阶段：真实 KuaiRec 数据链路已跑通，进入实验协议与生成式召回可信度修正阶段。

## 1. 项目定位

`CognID-GenRec` 是一个基于 KuaiRec 开源短视频数据集的多行为生成式推荐离线实验项目。当前主链路为：

```text
KuaiRec raw CSV
  -> schema/loaders
  -> watch_ratio 多行为标签
  -> 用户时序与 train/validation/test 字段
  -> item/user features
  -> Semantic ID
  -> Causal Transformer + InfoNCE
  -> item embedding / ANN index
  -> Top-K 分层评估
  -> FastAPI /recommend
```

旧版小样本内容推荐代码继续作为 historical baseline 和回归测试路径，不代表当前 KuaiRec 主实验。

## 4. 2026-07-11 项目审计结论

### 4.1 当前方法表只能视为诊断结果

KuaiRec 官方将 small matrix 定义为密度 99.6% 的 fully-observed 矩阵：1,411 个用户、3,327 个视频、4,676,570 条交互；big matrix 为 7,176 个用户、10,728 个视频、12,530,806 条交互、密度 16.3%。官方说明见 [KuaiRec README](https://github.com/chongminggao/KuaiRec)。

本项目当前把 small matrix 按时间排序后用于 leave-one-out next-item 评估。只读审计得到：

- Popular/ItemCF 的候选来源是 sequence catalog，共 3,327 个视频；每个用户排除 train history 后平均只剩 14.63 个候选。
- ANN artifact 包含 10,731 个视频；排除同一份 history 后平均仍有 7,418.63 个候选。
- 在前一种候选规模下，若目标在剩余候选中均匀分布，随机 Top-10 的期望 HR 约为 0.7307；当前 Popular HR@10 为 0.7123。
- 将 Popular/ItemCF 改为只使用 train history 拟合，并让 test history 包含 validation 行为后，Popular HR@10 为 0.7073，ItemCF HR@10 为 0.9901。ItemCF 高分主要来自 fully-observed 候选饱和，而不是模型效果足以接近完美。

因此当前 Popular、ItemCF、ANN Transformer 不处于统一候选空间，现有指标不能用于判断模型优劣，也不能作为简历中的模型提升结论。

代码层现已具备统一候选空间的重跑入口，但上述数字来自旧产物，尚未用真实 big matrix 和新 manifest 重新生成，因此结论边界暂不改变。

### 4.2 训练与推理尚未形成同一条模型链路

`scripts/train_kuairec_retriever.py` 会训练并保存 `SequenceEncoder`，但 `scripts/evaluate_kuairec_retriever.py` 和 `src/cognid_genrec/service/kuairec_api.py` 当前没有恢复该 encoder。两处均使用历史 item embedding 的平均值作为 ANN query。

当前 `ann_transformer` 更准确的含义是“使用训练产出的 item embedding 做均值查询的 ANN 诊断路径”，不能写成 Transformer user encoder 的真实离线效果。

### 4.3 长序列行为与时间特征存在对齐风险

训练数据构建时，item history 截取最后 `max_len` 个事件，但 action/time delta 按序列开头切片。KuaiRec small matrix 的平均历史长度超过 3,300，远大于默认 `max_len=64`，所以当前训练中的 item、action、time 可能不来自同一批事件。

### 4.4 Semantic ID 仍是实验性能力

当前 KMeans 实现在每一层独立聚类同一份 TF-IDF 特征，并非 residual quantization。10,731 个 item 最终只有 198 个组合 ID，冲突率为 98.15%。这组 ID 可以用于主题簇观察，但不能直接作为一一对应的生成式 item token。

后续需要借鉴 GRID/GenRec/OpenOneRec 的 residual quantization、冲突消歧，以及 SID/PID 双口径评估：

- [GRID](https://github.com/snap-research/GRID)
- [GenRec](https://github.com/phonism/genrec)
- [OpenOneRec](https://github.com/Kuaishou-OneRec/OpenOneRec)

### 4.5 Evaluation Protocol V2 已完成第一阶段

- prepare 阶段生成 `candidate_manifest.json`，记录协议名、matrix、候选全集、full-sort 模式、字段边界、seed 和 schema version。
- `user_sequences.jsonl` 新增 `test_history_item_ids`，明确 test history 可以观察 validation 事件，但不能包含 test target。
- Popular、ItemCF、ANN 评估共享同一候选清单；ANN artifact 缺少协议候选时直接失败。
- Popular/ItemCF 通过 `train_only` 视图拟合，不再读取 held-out item；Coverage 使用协议候选全集作为分母。
- 评估入口会拒绝 target 不可达、split 越界、历史项泄漏、候选越界、重复项、用户集合或推荐数量不一致。

当前完成的是 `sequential_next_item` 协议基础设施，不等于完整 P0-1：正式 big-matrix 重跑、`fully_observed_preference`、数据指纹和真实新报告仍待完成。

### 尚未验证的边界

- 尚未在真实 KuaiRec 数据上按统一 candidate manifest 重跑三种方法；三方法共享候选空间目前只通过 tiny artifact 回归测试验证。

## 6. 当前实验与对外表述边界

- 所有结果均为 KuaiRec 开源数据上的离线实验，不代表线上 CTR、观看时长、留存或收入提升。
- 当前 ItemCF 近满分是诊断信号，不能表述为算法达到接近 100% 的真实推荐效果。
- 当前 ANN 指标没有使用训练好的 Transformer user encoder，不能作为 Transformer 模型效果。
- 当前 Semantic ID 冲突率过高，不能表述为已完成高质量生成式 item tokenization。
- FastAPI 是本地演示接口，不是生产级推荐服务。
